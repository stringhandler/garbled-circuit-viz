Garbled Circuit Visualizer

A garbled circuit lets two parties compute a function of their private inputs without either party revealing those inputs. Alice (the garbler) encrypts the circuit by replacing every gate's truth table with an encrypted “garbled table.” Bob (the evaluator) receives the garbled circuit and uses Oblivious Transfer to obtain exactly one label per input wire — enough to evaluate the circuit gate by gate, without learning any other values.

Each party knows only their own input bits — never the other party’s. Alice will send her labels directly to Bob (Phase 3). Bob will obtain his labels via Oblivious Transfer (Phase 4), a protocol that prevents Alice from learning which label Bob selected.

Known Wire Labels

One label per wire. Labels are random 128-bit values — indistinguishable from noise. The color bit (LSB of byte 0) encodes the bit value in this implementation: color 0 → value 0, color 1 → value 1. Bob reads this bit to learn the result.

Garbled Table (active gate)

Bob picks the row using c(a)·2 + c(b), where c(x) is the color bit (LSB) of each input label. He decrypts that row with AES-Davies-Meyer using both input labels as keys. The other rows produce garbage — he can decrypt exactly one.

Oblivious Transfer

A 3-message protocol (Chou-Orlandi 2015 over Ristretto255) that transfers exactly one of Alice’s two labels to Bob. Alice never learns Bob’s selection bit. Bob never learns the label he didn’t receive. Each of Bob’s input wires requires one OT round.

Protocol Log
Who Learns the Output?

Bob ends up holding one label per output wire. Who learns the actual bit values depends entirely on who holds the label→bit mapping.

Option 1 · Bob Learns Standard Yao

Alice provides a decoding table mapping each output label to 0 or 1. Bob translates his labels and learns the result. Alice learns nothing.

In this visualizer the color bit (LSB of each label) directly encodes the bit value, so Bob reads it without a separate decoding table — a common simplification.

Option 2 · Alice Learns

Alice does not provide the decoding table. Bob sends his output labels back to Alice. She checks which label she originally assigned to 0 vs 1 and learns the result. Bob learns nothing beyond what the output implies.

Option 3 · Both Learn

Alice publishes the decoding table so Bob can decode, then communicates the result back. Alternatively, output wires can be assigned to different parties — some decode to Alice, some to Bob.

Big-picture rule: whoever knows the output label mapping learns the output.
